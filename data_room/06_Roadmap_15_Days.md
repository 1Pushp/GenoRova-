# GenorovaAI - 15-Day Roadmap

Date: April 17, 2026

This roadmap assumes focused evening work and keeps the story aligned with the
current prototype stage.

## Days 1-3: Diligence Consistency And Demo Readiness

Day 1:

- finalize the data-room docs
- standardize wording across README, reports, and demo script
- deliverable: one coherent diligence narrative

Day 2:

- capture core product screenshots
- verify demo prompts and backup prompts
- deliverable: screenshot set and clean demo flow

Day 3:

- verify Render environment variables and deployment path
- confirm auth, chat, stats, and score routes in the deployed configuration
- deliverable: deployment checklist and route smoke-check record

## Days 4-6: Product Reliability

Day 4:

- replace deprecated FastAPI startup hooks with lifespan handlers
- deliverable: cleaner startup path with reduced warnings

Day 5:

- add stronger persistence planning for user history and reports
- define the storage path for durable user-bound artifacts
- deliverable: documented persistence design

Day 6:

- add basic monitoring, structured logs, and failure reporting
- deliverable: minimum ops visibility for demos and advisor access

## Days 7-9: Scientific Evidence Hygiene

Day 7:

- audit older scientific reports for conflicting target and docking language
- retire or relabel unsupported claims
- deliverable: cleaned scientific evidence set

Day 8:

- create a compact benchmark summary for current model behavior
- separate exact metrics from heuristic signals
- deliverable: one-page benchmark appendix

Day 9:

- define the next generation-quality experiment plan
- lock seeds, checkpoints, and reporting format
- deliverable: reproducible experiment plan

## Days 10-12: Advisor And Faculty Review Readiness

Day 10:

- prepare an advisor packet with architecture, limitations, and ask list
- deliverable: advisor review bundle

Day 11:

- prepare a faculty-safe product walkthrough
- deliverable: presentation deck outline and supporting notes

Day 12:

- collect first-pass feedback and convert it into issue tickets
- deliverable: ranked feedback backlog

## Days 13-15: Follow-Through And Polish

Day 13:

- address the highest-risk documentation and product feedback
- deliverable: tighter demo and clearer trust messaging

Day 14:

- re-run tests and frontend build after cleanup
- deliverable: refreshed verification log

Day 15:

- produce the next diligence snapshot for outreach or review
- deliverable: versioned data-room refresh

## Roadmap Outcome

At the end of this 15-day window, the goal is not to claim scientific
breakthrough. The goal is to present GenorovaAI as a disciplined,
well-documented computational platform with honest limitations, a stable demo,
and a credible path to stronger evidence.
