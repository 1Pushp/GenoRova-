# Screenshot Checklist

Date: April 17, 2026

Capture only screens that match the current honest product story: protected
workspace, evidence-weighted computational analysis, and explicit limitations.

## Capture Order

1. `01_auth_entry.png`
- show the login or signup screen
- keep the research-support framing and trust note visible
- use a clean demo account name if the signed-in user is shown elsewhere

2. `02_guided_workspace_home.png`
- sign in and capture the empty guided workspace
- include the "Recommended first demo" area
- keep the trust note and example prompt cards visible

3. `03_ranked_candidate_set.png`
- run: `Show the top computational candidates in the active diabetes workflow`
- capture the top ranked candidate plus the ranked candidate set section
- make sure the trust language is still visible on screen

4. `04_validation_ledger.png`
- stay on the ranked-candidate response
- scroll so the validation ledger is readable
- include SA, novelty, comparator delta, docking mode, and final decision if visible

5. `05_plain_language_explanation.png`
- run: `Explain the best molecule simply`
- capture the explanation summary and candidate card together if possible

6. `06_follow_up_optimization.png`
- run: `Optimize the best molecule for oral delivery`
- show the follow-up reasoning and limitations, not just the prompt

7. `07_known_molecule_scoring.png`
- run: `Score this molecule: CCO`
- capture the simple known-molecule scoring response for a lower-friction example

8. `08_session_persistence.png`
- refresh while logged in
- capture the workspace still authenticated after refresh

## Optional Supporting Screens

9. `09_health_or_ops_status.png`
- only if needed for advisor diligence
- capture `/health` or `/ops/status` in a browser tab
- use this only in diligence materials, not as a headline product image

10. `10_html_report.png`
- optional
- capture the HTML report only if the visible section contains caveats and no stale claims

## Capture Rules

- use one browser theme and one browser width throughout
- prefer desktop-width captures unless a mobile view is specifically needed
- hide private email addresses where possible
- do not crop out trust or limitation text if it is visible near the captured result
- do not capture stale legacy screens from inactive components
- do not imply novelty, clinical validation, or production readiness

## Suggested Caption Style

- "Protected workspace entry"
- "Guided first-run workspace"
- "Ranked candidate review"
- "Validation ledger"
- "Plain-language explanation"
- "Follow-up optimization reasoning"
- "Known-molecule scoring"
- "Authenticated session persistence"
